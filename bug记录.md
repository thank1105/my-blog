# Bug 记录

> 用途：记录已发生的问题、根因、修复方式和预防措施，便于后续复盘，避免同类错误重复出现。

## BUG-2026-07-21-001：文章上传封面图后保存失败

### 基本信息

- **发现时间**：2026-07-21
- **影响页面**：`/admin/articles/[id]/edit`
- **影响功能**：文章编辑 / 新建文章时上传封面图并保存
- **严重程度**：中
- **状态**：已修复并推送
- **修复提交**：`16e756a fix(content): accept local cover image uploads`

### 问题现象

在文章编辑页面上传封面图后，上传接口本身返回成功，页面也能显示封面预览；但是点击“保存”时提示：

```text
表单校验失败
```

文章内容无法保存。

上传接口返回的封面图地址形如：

```text
/uploads/2026-07/<hash>.png
```

### 根因分析

封面图上传到本地 `public/uploads` 目录后，接口返回的是站内相对 URL（例如 `/uploads/2026-07/example.png`）。

但是服务端文章 Schema 使用了严格的 Zod `.url()` 校验：

```ts
coverImage: z.string().trim().url("封面图需为 URL")
```

Zod 的 `.url()` 不接受 `/uploads/...` 这种站内相对路径，因此服务端 Action 返回了表单校验错误。

另外，文章表单此前没有把服务端返回的 `fieldErrors` 映射回具体字段，导致用户只能看到笼统的“表单校验失败”，无法直接定位到“封面图”字段。

### 修复内容

1. 新增共享封面图校验：`src/lib/media.ts`。
2. 封面图现在允许以下两种来源：
   - 本地上传路径：`/uploads/...`
   - 外部图片地址：`http://...` 或 `https://...`
3. 文章表单、文章服务端 Schema 使用同一套封面图校验规则。
4. 作品和相册服务端 Schema 同步修复，避免同一个上传组件在其他内容类型中再次出现相同问题。
5. 文章表单现在会把服务端字段错误显示在对应字段下方。
6. 新增 `src/lib/media.test.ts`，覆盖本地路径、HTTP(S) 地址和非法地址。

### 验证结果

- `pnpm typecheck`：通过
- `pnpm lint`：无错误；3 个 warning 为仓库中原有 warning
- `pnpm test`：13 个测试文件、143 个测试全部通过
- 已确认 `origin/main` 指向修复提交 `16e756a`

### 预防措施

以后新增文件上传或媒体字段时，必须同时确认以下内容：

- [ ] 上传接口实际返回值的格式（相对路径、绝对 URL 或其他格式）
- [ ] 客户端 Schema 与服务端 Schema 是否接受同一种格式
- [ ] 新增上传字段是否有“上传成功后再提交”的集成验证
- [ ] 服务端 `fieldErrors` 是否能显示到具体表单字段
- [ ] 至少增加一个本地上传路径的 Schema 测试
- [ ] 如果多个内容类型共用上传组件，应优先抽取共享校验规则，避免只修复单一页面

### 复盘结论

上传组件的返回格式和服务端表单校验规则没有建立统一契约，导致“上传成功但保存失败”。今后媒体上传功能必须以共享 Schema 和测试作为接口契约，不能只验证上传接口自身返回 200。
