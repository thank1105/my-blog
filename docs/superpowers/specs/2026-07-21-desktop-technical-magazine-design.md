# 小川记事桌面端技术杂志风设计规格

日期：2026-07-21

## 目标

在现有“技术文章 + 两级专栏”信息架构上，优先完成桌面浏览器体验。解决当前页面色彩单一、视觉重点不足，以及文章封面与文字比例失衡的问题。

设计借鉴 Poetize 的色彩层次和内容氛围，但不复制其多模块个人主页结构。最终仍应是一眼可浏览、可筛选、可连续阅读的技术博客。

## 本轮范围

本轮完成：

- 桌面端首页的技术杂志风视觉升级。
- 首页、专栏文章页、标签页和归档页共用的文章列表升级。
- 桌面端专栏侧栏、搜索、标签、列表状态和分页视觉升级。
- `/columns` 专栏目录页视觉升级。
- Header、Footer 和文章详情关键区域的配色统一。
- 文章封面强制要求，包括数据模型、后台表单和服务端校验。
- 常见桌面宽度的浏览器检查，重点为 1440px 和 1920px。

本轮不完成：

- 平板布局和专项适配。
- 手机布局和专项适配。
- 移动端菜单、移动端专栏折叠和触摸交互优化。
- 非文章模块的视觉重构。

桌面 Web 是后续功能与视觉工作的首要交付面。平板与手机适配在桌面体验稳定后单独规划。

## 视觉方向

采用“克制多彩的技术杂志”方向。

### 色彩

- 页面底色：暖灰 `#F6F5F2`。
- 内容表面：白色 `#FFFFFF`。
- 主要文字：深蓝黑 `#172033`。
- 次要文字：中性灰蓝。
- 品牌主色：番茄橙 `#E85A2C`。
- 辅助色一：靛蓝 `#4F46E5`。
- 辅助色二：青绿 `#0F9F92`。

番茄橙用于品牌、主要交互和选中状态。靛蓝与青绿只用于专栏标记、数据强调和小面积浅色背景，不能与主色争夺注意力。Markdown 正文保持中性阅读环境。

### 质感

- 使用浅色分区、细边框和留白建立层级。
- 不使用厚重阴影、玻璃拟态或高饱和大面积渐变。
- 保持思源黑体和 JetBrains Mono 的现有字体分工。
- 圆角适中，文章列表不做悬浮卡片堆叠。
- 动效仅限封面轻微缩放、标题变色和状态反馈。

## 首页结构

首页保留“专栏侧栏 + 文章流”的双栏结构。

### 顶部导语

- 使用紧凑的编辑导语区，不恢复摄影大 Hero。
- 展示站点主张、简短说明、文章数量和主要专栏入口。
- 通过暖色浅底、番茄橙强调和少量辅助色建立第一视觉焦点。
- 导语不能占据超过首屏约三分之一高度，文章仍需在首屏可见。

### 左侧导航

- 搜索框与专栏树放入同一个有浅底色的侧栏面板。
- 当前专栏使用番茄橙突出。
- 一级、二级专栏通过缩进、字重和辅助色标记建立层级。
- 常用标签保留在侧栏下部，颜色有规则地轮换，但保持低饱和背景。
- 桌面端侧栏继续吸附在视口顶部附近。

### 文章流

- 列表顶部显示当前筛选结果数量和清除筛选入口。
- 每篇文章使用独立白色内容块，依靠间距、细边框和浅色状态区分。
- 空结果、加载和错误状态使用统一的浅色面板，不使用空白页面。

## 文章列表项

### 比例

- 桌面端采用左图右文。
- 封面约占列表项内容宽度的 32%，文字约占 68%。
- 推荐封面显示宽度为 300–340px，具体随主内容宽度调整。
- 封面固定为 `16:10`，源图建议不低于 `1200 × 750`。
- 图片使用 `object-fit: cover` 居中裁切，不拉伸。
- 图文间距保持紧凑，避免当前封面远离标题的大片空白。

### 内容层级

从上到下依次为：

1. 专栏路径与最多两个标签。
2. 最多两行的文章标题。
3. 最多两行的摘要。
4. 发布日期、阅读时长和真实阅读量。

专栏路径与标签使用番茄橙、靛蓝、青绿的小面积浅色标记。标题是列表项的第一文字焦点，摘要和元信息保持克制。

### 交互

- 整个封面和标题均可进入文章详情。
- 悬停时封面缩放幅度不超过 2%，标题变为番茄橙，边框轻微加深。
- 键盘焦点使用清晰的番茄橙描边。
- 不使用明显上浮、摇晃或循环动画。

## 封面强制规则

所有文章必须拥有真实封面，不生成装饰性默认封面。

- `Article.coverImage` 改为数据库非空字段。
- 创建、更新和自动保存的服务端输入要求非空封面。
- 后台新建文章时，没有封面不能保存草稿或发布。
- 编辑文章时允许替换封面，但不能将封面单独删除为空。
- 上传区显示“必填”，预览区域按 `16:10` 展示。
- 前台列表移除“无封面时隐藏图片区”的版式分支。
- 本地数据库按当前未上线假设重置，seed 中每篇示例文章提供真实封面。

校验必须同时存在于前端表单、服务端输入和数据库模型，避免绕过表单写入无封面文章。

## 其他页面

### 专栏目录 `/columns`

- 一级专栏使用有色顶部标记或浅色标题区。
- 二级专栏使用紧凑列表块展示名称和文章数。
- 三种强调色按稳定顺序使用，不能每次渲染随机变化。
- 保持两栏桌面结构和清楚的父子关系。

### 专栏、标签与归档

- 继续复用统一的文章列表组件。
- 页面标题区允许使用与当前筛选对应的小面积辅助色。
- 图文比例、标签样式、状态和分页必须与首页一致。

### 文章详情

- 保持现有舒适正文宽度、Markdown 和代码高亮。
- 强化顶部封面、专栏路径和标签区域。
- 正文背景保持白色或中性浅色，不使用彩色大底。
- 相关文章使用同一文章列表项的紧凑变体。

### Header 与 Footer

- 保留现有信息架构，不增加 Poetize 式大量菜单入口。
- Header 增强当前导航状态和搜索入口的品牌色反馈。
- Footer 使用暖灰分区，与正文形成明确结束区域。

## 组件边界

- `ArticleListItem`：负责统一图文比例、内容层级和交互状态。
- `ArticlesList`：负责桌面双栏结构、筛选结果和列表状态。
- `ColumnTree`：负责专栏层级和当前状态，不承担文章查询。
- `TagCloud`：负责稳定的辅助色映射和标签导航。
- 首页导语组件：只负责站点主张与概览，不承载筛选逻辑。
- 封面上传组件：负责必填展示、上传、预览和替换，不承担文章提交。

这些组件继续复用现有服务端查询和路由边界，避免为视觉调整复制业务逻辑。

## 错误与边界状态

- 封面缺失：后台阻止保存并在上传区就地提示。
- 封面加载失败：前台保留固定比例区域并显示明确的加载失败状态，不显示伪造封面。
- 搜索无结果：显示当前关键词和清除筛选入口。
- 专栏无文章：保留专栏标题与说明，并显示明确空状态。
- 图片过宽或过高：统一由固定比例容器裁切。
- 长标题和长摘要：分别限制为两行，避免破坏列表高度。

## 桌面验收

- 在 1440px 与 1920px 宽度检查首页、专栏、标签、归档和文章详情。
- 对照用户提供的两张截图检查封面宽度、图文比例、卡片高度和间距。
- 检查搜索、专栏筛选、标签筛选、分页与清除筛选。
- 检查键盘导航、焦点状态、悬停和减少动态效果偏好。
- 检查有长标题、长摘要、不同颜色封面和图片加载失败时的稳定性。
- 单元测试覆盖封面必填、服务端拒绝空封面和文章列表渲染分支。
- 完成实现后运行 `pnpm test`、`pnpm typecheck`、`pnpm lint` 和 `pnpm build`。

平板与手机布局不属于本轮验收条件，也不应为了本轮桌面交付引入新的移动端交互。
